# DifyCRM 10 分钟演示指南

> 适用场景：向同事/客户介绍「飞书 + AI + 本地 CRM」获客营销闭环  
> 演示入口：飞书 @ CRM 机器人（LangBot → Dify `crm_main` → DifyCRM API → MySQL）  
> **CRM 新手请先读：** [CRM-入门与指令地图.md](./CRM-入门与指令地图.md)（功能、指令接龙、AI 与 MySQL/RAG 分工）

---

## 一、用一句话介绍这个系统

**DifyCRM 是一套以获客营销为核心的智能 CRM：销售在飞书里用自然语言对话完成线索录入、评分、转客户、跟进和分析；Dify 先把话翻译成业务动作，关键数据落在本地 MySQL。**

它不是「又一个聊天机器人」，而是：**飞书做入口，Dify 做 AI 编排，MySQL 做真源** 的一体化业务系统。

---

## 二、这个系统能干什么（按角色）

| 角色 | 在飞书里怎么说（示例） |
|------|------------------------|
| **市场/运营** | 「帮我看看各渠道获客效果」「各来源转化怎么样」 |
| **销售** | 「列出线索」「录入一条高意向线索…」「把这条线索转成客户」「跟进一下周总…」 |
| **管理者** | 「看一下销售漏斗」「我的面板/待办」 |
| **演示者** | 一条自然语言消息完成「线索进系统 → 自动评分 → 转商机 → 记跟进 → 看分析」 |

**当前已具备（演示可用）：**

- **自然语言 → CRM 动作**（Dify LLM 解析，无需记 `/指令`）
- 渠道、营销活动、线索的创建与查询
- 线索**自动评分**（来源、意向、预算、痛点关键词）
- 高意向线索**一键转客户 + 自动建商机**
- 跟进记录（含摘要与下一步建议日期）
- 获客分析、来源分析、销售漏斗、个人面板
- 数据**持久化**在 MySQL 库 `dify_crm`

**仍兼容：** 直接发 `/获客分析`、`/线索列表` 等斜杠指令（跳过 LLM 解析，更快）。

**刻意不做（本阶段）：** 复杂审批、多级权限、审计日志、独立 Web 前台、电子签章。

**与旧机器人关系：** 表格分析机器人独立运行；CRM 机器人走 `crm_main`，互不干扰。

---

## 三、架构（讲 30 秒即可）

```text
飞书用户 @ 机器人（自然语言）
    ↓ 长连接
LangBot
    ↓
Dify crm_main
    ├─ LLM：自然语言 → /指令
    └─ HTTP → DifyCRM API → MySQL dify_crm
```

**设计要点（可强调）：**

- 用户**说人话**，不用背命令；LLM 负责「听懂」，CRM API 负责「做对、存对」。
- 结构化数据（线索、金额、状态）只进 **MySQL**；话术/FAQ 后续进 Dify 知识库 RAG。
- 业务规则集中在 **DifyCRM API**，不是多个系统拼凑。

## 五、10 分钟演示脚本（推荐节奏）

| 时间 | 环节 | 你对观众说什么 | 飞书里发什么（自然语言） |
|------|------|----------------|--------------------------|
| **0:00–1:30** | 开场 | 强调「说人话就能用 CRM」，不用记命令；数据落 MySQL | 「你能帮我做什么？」或「有哪些功能？」 |
| **1:30–3:00** | 看渠道 | 市场最关心哪条线出线索、质量如何 | `帮我看看各渠道获客效果` |
| **3:00–4:30** | 看线索池 | 销售按评分优先跟进 | `列出当前线索，按评分排序` |
| **4:30–6:30** | 现场录入 | 一条消息进系统并自动打分 | `帮我录入一条线索：周总，云启科技，来源官网表单，高意向，预算20万，痛点是线索分散销售跟进不及时` |
| **6:30–7:30** | 转化 | 高意向转客户并建商机 | `把线索4转成客户`（ID 以列表为准，演示前确认） |
| **7:30–8:30** | 跟进 | 沟通留痕 + 下一步提示 | `跟进一下周总，他下周想看获客营销演示，可以安排飞书会议` |
| **8:30–9:30** | 管理视角 | 漏斗与面板 | `看一下销售漏斗` → 可选 `打开我的面板` |
| **9:30–10:00** | 收尾 | 架构 + 后续（飞书看板、RAG 话术库）；Q&A | — |

> **技巧：** 每条话**一条消息**发完；群聊 **@ 机器人**。Dify 调试可在结束节点看 `parsed_command` 确认 LLM 解析结果。

## 六、逐步演示稿（复制到飞书）

### 步骤 1：开场 / 能力介绍（可选）

```text
你能帮我做什么？
```

或：

```text
有哪些 CRM 功能？
```

**预期：** 返回帮助菜单或功能说明（解析为 `/help` 一类）。

**讲解词：** 「不用记命令，像跟同事说话一样用 CRM。」

---

### 步骤 2：获客渠道分析

```text
帮我看看各渠道获客效果
```

**预期：** 各渠道线索数、转化数、成本、平均分；末尾有「建议」。

**讲解词：** 「数据来自 MySQL 真实业务表，不是 LLM 编出来的。」

---

### 步骤 3：线索列表

```text
列出当前线索，按评分从高到低
```

**预期：** 分段展示线索，「张明」通常排前面。

**讲解词：** 「系统自动评分：来源、意向、预算、痛点。」

---

### 步骤 4：新建线索（现场感最强）

```text
帮我录入一条线索：周总，云启科技，来源官网表单，高意向，预算20万，痛点是线索分散、销售跟进不及时，希望在飞书里统一管理
```

**预期：** 返回 ID、公司、来源、评分及原因。

**讲解词：** 「LLM 把自然语言变成创建线索动作，字段写入 MySQL。」

---

### 步骤 5：线索转客户

先 `列出当前线索` 确认周总的 **ID**（例如 4），再发：

```text
把线索4转成客户
```

或：

```text
帮我把周总这条线索转成客户
```

**预期：** 已创建客户与商机。

---

### 步骤 6：记录跟进

```text
跟进一下周总，客户希望先看获客营销模块，下周可以安排飞书会议演示
```

**预期：** 跟进 ID、摘要、下一步建议、建议日期。

---

### 步骤 7：漏斗与面板

```text
看一下销售漏斗
```

可选：

```text
打开我的面板，看看待办和高分线索
```

**预期：** 漏斗阶段分布 + 解读；或面板汇总。

---

## 七、30 秒「电梯演讲」

> 我们在飞书里接了 CRM 助手，**直接说人话**就行。市场问「各渠道获客怎么样」，系统自动查库分析；销售说「录入一条高意向线索」，系统打分入库；说「转成客户」「跟进一下」，完整走线索到商机链路；管理者看漏斗。Dify 负责听懂人话，MySQL 存真数据，和表格分析机器人分开各干各的。

## 八、常见问题（演示时可能被问）

| 问题 | 简短回答 |
|------|----------|
| 必须记 `/指令` 吗？ | **不用**。自然语言为主；斜杠指令仍可用，适合熟练用户。 |
| AI 在哪？ | Dify **LLM 把自然语言解析成 CRM 动作**；评分、跟进摘要等规则在 CRM API；话术库 RAG 是下一步。 |
| 数据存在哪？ | MySQL `dify_crm`；LLM 不编造线索数和金额。 |
| 和飞书多维表格？ | MySQL 是真源；飞书适合看板协作，后续可同步展示。 |
| 和表格分析机器人？ | 不冲突，两个流水线、两个 Dify 应用。 |

## 九、演示翻车应急

| 现象 | 处理 |
|------|------|
| 机器人无回复 | `docker logs langbot`；确认 Dify、LangBot、CRM API |
| 一直返回帮助、答非所问 | Dify 看 `parsed_command` 是否解析错；换更直白说法；或临时用 `/获客分析` |
| LLM 节点报错 | Dify 里检查「自然语言转指令」节点模型是否已配置（如 DeepSeek） |
| 超时 | 确认 `http://127.0.0.1:5055/health` |
| 转客户 ID 不对 | 先说「列出线索」确认 ID |
| 恢复演示数据 | `G:\AIstudy\DifyCRM` 执行 `.\scripts\init_db.ps1` |
